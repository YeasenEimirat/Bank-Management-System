CURRENCY SEARCH FIX

Problem:
The JavaFX search field called Currency.findByCode() only. Entering a country
name such as Algeria therefore returned "Currency not found" even though the
country existed in Currencies.txt.

Fix:
- Added Currency.find(String), which accepts either a country name or currency code.
- Updated JavaFX to call the combined search method.
- Search now ignores leading/trailing spaces and letter case.
- Updated the search-field hint to show both accepted input types.

Verified examples:
- Algeria -> DZD / Algerian Dinar
- DZD -> Algeria / Algerian Dinar
- "  algeria  " -> DZD / Algerian Dinar
